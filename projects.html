RAG PDF Chatbot

This project builds an end-to-end Retrieval-Augmented Generation (RAG) pipeline that allows users to upload any PDF document and ask natural language questions about its content. The system retrieves the most relevant sections from the document and uses a large language model to generate accurate, context-grounded answers.

The pipeline processes PDFs into 500-word overlapping chunks and stores them as 384-dimensional semantic vectors using HuggingFace sentence-transformers (all-MiniLM-L6-v2). When a user asks a question, the system performs semantic search using FAISS to retrieve the top-3 most relevant chunks, even when the user's wording differs significantly from the document text.

The retrieved context is passed to Google Gemini 2.5 Flash API with a custom prompt engineering template that instructs the model to answer only from the provided context — preventing hallucination and keeping answers grounded in the source document.

⚡ Key Achievements

  • Processes PDFs into 500-word overlapping chunks for better context retention
  • Stores 384-dimensional semantic vectors in FAISS for fast retrieval
  • Retrieves top-3 relevant chunks per query using cosine similarity
  • Achieved correct answers on factual queries across 15-page research documents
  • Integrated Gemini 2.5 Flash API with custom prompt engineering

Tech Stack

Python LangChain FAISS HuggingFace Gemini API PyPDF2 Sentence Transformers

What I Learned

Building this project gave me deep hands-on experience with vector databases, semantic search, and prompt engineering — skills that are increasingly central to production AI systems. I learned how chunking strategy significantly impacts retrieval quality.

Multi-Agent Atari Boxing DQN

This project trains two competing Deep Q-Network (DQN) agents to play the Atari Boxing game against each other using the PettingZoo multi-agent environment and PyTorch. The agents learn entirely through self-play — no hardcoded rules or human demonstrations.

Each agent uses a CNN-based DQN architecture consisting of 3 convolutional layers followed by 2 fully connected layers, operating on 4 stacked 84×84 grayscale frames as input. The stacking of frames gives the agent temporal information — it can infer velocity and direction of movement from consecutive frames.

Key RL techniques implemented from scratch include Experience Replay with a 10,000-sample buffer, Target Networks for stable Q-value updates, and Epsilon-Greedy exploration decaying from 1.0 to 0.1 over training. The model was trained over 200 episodes on a T4 GPU using Google Colab.

⚡ Key Achievements

  • Achieved 62%/38% win-rate split between agents after 200 episodes
  • Trained on T4 GPU via Google Colab with full experiment tracking
  • Implemented Experience Replay (10K buffer) and Target Networks from scratch
  • Preprocessed 210×160×3 RGB frames to 84×84 grayscale using OpenCV
  • Visualized reward curves and win-rate progression using Matplotlib

Tech Stack

Python PyTorch PettingZoo OpenCV Matplotlib Google Colab CUDA (T4 GPU)

What I Learned

This project gave me practical experience with the full DQN training loop, the importance of experience replay for stability, and how multi-agent environments introduce non-stationarity that makes training more challenging than single-agent RL.

Loan Default Prediction with MLflow

This project builds a complete end-to-end ML pipeline for predicting loan defaults using 32,581 real loan records. Four different machine learning models were trained and compared — Logistic Regression, Decision Tree, Random Forest, and XGBoost — with all experiments tracked using MLflow.

The pipeline covers the full data science lifecycle: data cleaning, feature engineering, class imbalance handling using SMOTE, hyperparameter tuning, model evaluation, and experiment tracking. Each experiment run is logged to MLflow with metrics, parameters, and artifacts for full reproducibility.

XGBoost emerged as the best performing model, achieving 93.4% accuracy and an AUC of 0.947. Feature importance analysis identified the top predictors of loan default, providing actionable insights for credit risk assessment.

⚡ Key Achievements

  • Trained on 32,581 real loan records from a public credit dataset
  • XGBoost achieved 93.4% accuracy and AUC of 0.947
  • All 4 models tracked via MLflow with full experiment logging
  • Handled class imbalance using SMOTE oversampling
  • Identified top loan default predictors via feature importance analysis

Tech Stack

Python XGBoost MLflow Scikit-Learn Pandas SMOTE Matplotlib

What I Learned

This project taught me the importance of experiment tracking in ML — without MLflow, comparing 4 models across multiple hyperparameter configurations would have been chaotic. I also gained practical experience with handling real-world class imbalance problems.

DataPulse Dashboard

DataPulse is a fully interactive, dark-themed analytics dashboard built with D3.js v7 that visualizes over 11,900 records across three different datasets — no backend required. All data is loaded client-side using the Fetch API, making it a pure front-end application.

The dashboard features 7 chart types including bar charts, line charts, scatter plots, heatmaps, area charts, pie charts, and KPI cards — all built from scratch using D3.js SVG rendering. Every chart is interactive with tooltips, hover states, and click-to-filter functionality.

The three datasets cover 1,000 e-commerce orders, 7,300 climate readings, and 3,600 stock records — demonstrating the dashboard's versatility across different data domains and chart requirements.

⚡ Key Achievements

  • Built 7 chart types from scratch using D3.js v7 SVG rendering
  • Visualizes 11,900+ records across 3 different datasets
  • Debugged 4 critical rendering bugs independently through systematic isolation
  • Zero backend — pure front-end with client-side data loading
  • Full dark theme with interactive tooltips and hover states on all charts

Tech Stack

D3.js v7 JavaScript Python Pandas HTML/CSS Fetch API

What I Learned

Building complex visualizations with D3.js from scratch taught me how SVG rendering works at a low level, and how to think about data binding, scales, and axes. Debugging rendering issues without a framework forced me to deeply understand the DOM.

Customer Shopping Behavior Analysis

This is an end-to-end data analytics project analyzing 3,900 retail transactions to uncover customer purchasing patterns, segment performance, and revenue distribution. The project covers the full analytics pipeline from raw data ingestion and SQL analysis to interactive Power BI visualization.

The SQL analysis uses 10 complex queries including Common Table Expressions (CTEs) and window functions to calculate running totals, rank customers by lifetime value, and identify seasonal purchase trends. All queries were written and executed in PostgreSQL.

The final Power BI dashboard reveals key business insights including a $157K vs $75K gender revenue split and identifies Young Adults as the top customer segment with $62,143 in total spend — actionable findings that could directly inform marketing strategy.

⚡ Key Achievements

  • Analyzed 3,900 retail transactions end-to-end
  • Wrote 10 SQL queries using CTEs and window functions
  • Uncovered $157K vs $75K gender revenue split
  • Identified Young Adults as top segment with $62,143 total spend
  • Built interactive Power BI dashboard for business stakeholders

Tech Stack

This project reinforced how much insight can come from structured data when the right questions are asked. Writing complex SQL with window functions taught me to think about data transformation as a pipeline rather than individual queries.
